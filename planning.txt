===========================================================
PLANEJAMENTO DO PROJETO: SISTEMA DE CAPTURA DE LEADS
===========================================================

1. OBJETIVO
-----------------------------------------------------------
API robusta para captura de contatos em eventos, com foco
em integridade, segurança e escalabilidade (Node.js + Prisma 7).

2. TECNOLOGIAS (STACK)
-----------------------------------------------------------
- Runtime: Node.js (v20+)
- Linguagem: TypeScript
- Banco: PostgreSQL
- Framework: Express
- Driver/Adapter: pg + @prisma/adapter-pg
- ORM: Prisma 7
- Validação: Zod
- Segurança: Helmet, CORS, Bcryptjs, JWT
- Imagens: Multer + Sharp

3. MODELAGEM DETALHADA DO BANCO DE DADOS
-----------------------------------------------------------

Tabela: Users (Usuários do Sistema)
 - id: UUID (Primary Key, Default: uuid_generate_v4())
 - name: String
 - email: String (Unique)
 - password: String (Hash)
 - role: Enum (ADMIN, USER)
 - avatar_url: String (Optional)
 - created_at: DateTime (Default: now())
 - updated_at: DateTime (Updated_at)

Tabela: Events (Eventos/Feiras)
 - id: UUID (Primary Key)
 - name: String
 - location: String (Optional)
 - start_date: DateTime
 - end_date: DateTime
 - created_at: DateTime (Default: now())

Tabela: Products (Catálogo de Interesses)
 - id: UUID (Primary Key)
 - name: String (Unique - ex: 'TriStat', 'Premier Hb9210')
 - description: Text (Optional)

Tabela: Leads (Contatos capturados)
 - id: UUID (Primary Key)
 - name: String
 - email: String (Optional)
 - phone: String
 - company: String (Antiga 'instituicao')
 - city: String
 - state: String (Char 2)
 - business_card_url: String (Caminho da foto do cartão)
 - observations: Text (Optional)
 - status: Enum (NEW, CONTACTED, QUALIFIED, LOST)
 - user_id: UUID (Foreign Key -> Users)
 - event_id: UUID (Foreign Key -> Events)
 - created_at: DateTime (Default: now())
 - updated_at: DateTime

Tabela: _LeadInterests (Relação Muitos-para-Muitos)
 - A: Lead_ID (Foreign Key)
 - B: Product_ID (Foreign Key)

4. FLUXO DE IMAGENS
-----------------------------------------------------------
1. Upload via Multer (Pasta temp)
2. Processamento via Sharp (Redimensionar/Comprimir)
3. Salvar em /uploads
4. Gravar o caminho (string) no banco de dados.

5. ARQUITETURA DE PASTAS
-----------------------------------------------------------
/src
  /routes      -> Definição de rotas
  /controllers -> Lógica de entrada/saída (Express)
  /services    -> Regras de negócio e Prisma
  /middlewares -> Auth, Error Handler, Zod Validation
  /config      -> Configuração do Driver PG e Multer

6. DIFERENCIAIS DE IMPLEMENTAÇÃO
-----------------------------------------------------------
- Uso de UUID: IDs não sequenciais para maior segurança.
- Central Error Handler: Middleware para capturar erros de forma global.
- Normalização: Separação de Eventos e Produtos para relatórios precisos.
- Otimização: Uso do Sharp para reduzir o peso das fotos de cartões de visita.

7. DEFINIÇÃO DE ENDPOINTS (API ROUTES)
-----------------------------------------------------------

AUTENTICAÇÃO & USUÁRIOS (Auth)
- POST   /auth/login          -> Autentica usuário e retorna JWT Token.
- POST   /users               -> Cadastro de novos usuários (Restrito: Admin).
- GET    /users/me            -> Retorna dados do perfil do usuário logado.
- PATCH  /users/:id.          -> Atualiza a foto de perfil do usuário.
- GET    /users               -> Cadastro de novos usuários (Restrito: Admin).
- GET    /users/:id           -> Cadastro de novos usuários (Restrito: Admin).
- DELET  /users/:id           -> Cadastro de novos usuários (Restrito: Admin).

EVENTOS (Events)
- GET    /events              -> Lista todos os eventos cadastrados.
- POST   /events              -> Cria um novo evento (Ex: Feira Hospitalar).
- GET    /events/:id          -> Detalhes de um evento específico.
- PUT    /events/:id          -> Atualiza dados do evento.

PRODUTOS (Products)
- GET    /products            -> Lista catálogo para seleção de interesses.
- POST   /products            -> Cadastra novo produto no sistema.
- DELETE /products/:id        -> Remove produto (se não houver vínculo).

LEADS (Captura de Contatos)
- POST   /leads               -> Cadastra novo lead (Vincula Evento, User e Interesses).
- GET    /leads               -> Lista geral de leads (com filtros e busca).
- GET    /leads/event/:id     -> Filtra leads capturados em um evento específico.
- GET    /leads/:id           -> Detalhes completos de um lead (inclui interesses).
- PATCH  /leads/:id/status    -> Altera status (Novo, Qualificado, Perdido, etc).
- POST   /leads/:id/card      -> Upload/Atualização da foto do cartão de visita.
- DELETE /leads/:id           -> Remove um lead do sistema.

7. PADRÕES DE RESPOSTA (HTTP STATUS CODES)
-----------------------------------------------------------
- 200 OK: Sucesso na leitura/atualização.
- 201 Created: Sucesso na criação.
- 400 Bad Request: Erro de validação (Zod).
- 401 Unauthorized: Token ausente ou inválido.
- 403 Forbidden: Usuário sem permissão (Role).
- 404 Not Found: Recurso não encontrado no banco.
- 500 Internal Server Error: Erro inesperado capturado pelo Global Handler.
===========================================================
